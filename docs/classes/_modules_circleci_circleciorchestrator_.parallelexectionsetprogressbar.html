Class ParallelExectionSetProgressBar

{@see ParallelExectionSetProgressBar} :

  • Displays a multi progress bar for a Parallel Execution Set
  • based on https://www.npmjs.com/package/cli-progress#multi-bar-mode

Each Progress Bar, notifies the progress status of one Pipeline Execution. As of Circle CI API v2, there are 3 possible states for a pipeline execution :

  • pending
    (pipeline was triggered and pipeline execution is running) 
  • errored
    (pipeline execution completed witjh 
    erros
    ) 
  • created
    (pipeline execution succcessfully completed, with no 
    erros
    )

Before a pipeline is triggered, the pipeline execution does not exists, as far as the Circle CI API is concerned. So I added a fourth one, in order to display the progress bar for all planed pipeline executions, before they even exists in the CircleCI API v2 :

  • untriggered
    (pipeline execution was not triggered and is running) 
  • pending
    (pipeline execution was triggered and is running) 
  • errored
    (pipeline execution completed witjh 
    erros
    ) 
  • created
    (pipeline execution succcessfully completed, with no 
    erros
    )

Also see 

barGlue
, to have custom text displayed with status'

comment

All Circle CI API calls are asynchronous, RxJS ObservableStreams, cf. https://github.com/gravitee-lab/GraviteeReleaseOrchestrator/issues/9
